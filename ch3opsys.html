Operating Systems Terminology

TERM

DEFINITION
operating system The software that controls the overall operation of the computer including file management, the user interface, and the program execution environment.
job queue a list of tasks to be carried out in turn by an operating system
FIFO First in first out. The objects are removed from a queue in the order in which they arrived.
interactive processing Operating system that allows a program being executed to carry on a dialogue with the user through remote terminals.
time-sharing Operating system that provides service to multiple users at the same time.
multiprogramming Time is divided into intervals and then the execution of each job is restriced to only one interval at a time.
application software Programs for performing tasks particular to the machine's utilization.
system software Performs tasks that are common to computer systems in general.
utility software Consists of software units that extend the capabilities of the operating system.
graphical user interface (GUI) Communicates with users using objects to be manipulated, such as files and programs, are represented pictorially on the display as icons.
shell Older user interface that communicates with users through textual messages using a keyboard and monitor screen.
kernel Internal part of an operating system. Contains software components that perform the very basic functions required by the computer installation.
device driver Software units that communicate with the controllers to carry out operations on the peripheral devices attached to the machine.
file manager Coordinates the use of the mahine's mass storage facilities
memory manager Coordinates the machine's use of main memory.
virtual memory Large fictional memory space created by paging.
boot strapping Procedure that transfers the operating system from mass storage into main memory.
ROM Small portion of a computer's main memory where the CPU expects to find its intitial program that is constructed from special nonvolatile memory cells.
deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
spooling Technique of holding date for output at a later but more convenient time.
super user (or administrator) Account established by a person. This person gains higly privileged access to the operating system.
multi-factor authentication proccess of more than one piece of evidence being required to gain access.
